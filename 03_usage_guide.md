## 1. 시스템 개요

이 시스템은 **어떤 분야든** 체계적인 학습 자료를 만들 수 있는 2단계 프롬프트 + Claude 프로젝트 시스템입니다.

```
[1단계] 커리큘럼 생성
    01_curriculum_prompt.md 사용
    → 결과물: 커리큘럼 문서 (.md)

[2단계] 프로젝트 세팅
    Claude 프로젝트 생성
    → 프로젝트 지침: 02_study_material_prompt.md
    → 프로젝트 파일: 커리큘럼 문서 첨부

[3단계] Chapter별 학습 자료 생성
    프로젝트 내에서 "Chapter N 학습 자료를 작성해주세요" 요청
    → 결과물: 각 Chapter 학습 자료 (.md)
```

**핵심 아이디어**: 커리큘럼(설계도)을 먼저 만들고, 프로젝트에 세팅해두면, 이후에는 Chapter 번호만 지정하면 일관된 형식의 학습 자료가 생성됩니다.

### 핵심 설계 원칙: 개념 폐쇄성 (Concept Closure)

이 시스템은 **"아직 배우지 않은 개념은 사용하지 않는다"**는 원칙 위에 설계되어 있습니다.

- **커리큘럼 프롬프트**에서 각 Chapter의 "도입 개념"과 "선수지식"을 명시하여, 어떤 시점에서 학습자가 어떤 개념을 알고 있는지 추적할 수 있는 지도를 만듭니다.
- **학습 자료 프롬프트**에서 이 지도를 참조하여, 해당 Chapter 시점에서 학습자가 알고 있는 개념의 범위 안에서만 자료를 작성하도록 강제합니다.

이를 통해, 학습자가 순서대로 자료를 읽을 때 "이게 뭐지? 어디서 나온 거지?" 하는 혼란 없이 자연스럽게 학습할 수 있습니다.

## 2. 전체 워크플로우

```
1️⃣ 주제 결정
   └─▶ "파이썬 프로그래밍을 공부하고 싶다"

2️⃣ 커리큘럼 생성 (일반 대화)
   └─▶ 01_curriculum_prompt.md의 내용 전체를 복사하여 전송
   └─▶ [학습 주제] 부분만 원하는 주제로 수정
   └─▶ 결과물: 커리큘럼 문서 저장 (.md)

3️⃣ 커리큘럼 검토 & 수정 ⭐ (품질의 핵심)
   └─▶ 같은 대화에서 수정 요청
   └─▶ 검토 체크리스트 활용 (3.3절 참조)
   └─▶ 최종 확정 후 .md 파일로 저장

4️⃣ Claude 프로젝트 세팅
   └─▶ 새 프로젝트 생성 (예: "파이썬 학습")
   └─▶ 프로젝트 지침(Instructions)에 02_study_material_prompt.md 내용 붙여넣기
   └─▶ 프로젝트 파일(Knowledge)에 커리큘럼 .md 파일 첨부

5️⃣ Chapter별 학습 자료 생성 (프로젝트 내 대화)
   └─▶ "Chapter 1의 학습 자료를 작성해주세요"
   └─▶ Chapter 하나당 하나의 대화 권장
   └─▶ 결과물: 각 Chapter 학습 자료 저장

6️⃣ 학습 자료 검토 & 보충
   └─▶ 같은 대화에서 보충/수정 요청
```

## 3. Step 1: 커리큘럼 생성

### 3.1 기본 사용법

1. **일반 대화** (프로젝트 밖)를 시작합니다.
2. `01_curriculum_prompt.md`의 내용 전체를 복사합니다.
3. `[학습 주제]` 부분(2곳: 첫 줄과 마지막 줄)을 원하는 주제로 바꿉니다.
4. Claude에게 전송합니다.

### 3.2 주제 범위 설정 팁

| 범위 | 예시 | 예상 Chapter 수 | 권장 여부 |
|------|------|-----------------|-----------|
| 너무 좁음 | "파이썬 for문" | 2~3개 | ❌ Section 수준의 주제 |
| 적절함 | "파이썬 프로그래밍" | 15~25개 | ✅ |
| 적절함 | "미시경제학" | 12~20개 | ✅ |
| 적절함 | "선형대수학" | 12~18개 | ✅ |
| 너무 넓음 | "컴퓨터 과학 전체" | 50개 이상 | ❌ 여러 Course로 분리 |

너무 넓은 주제는 여러 과정(Course)으로 나눠서 각각 커리큘럼을 만드세요.

### 3.3 커리큘럼 검토 체크리스트

커리큘럼이 생성되면 아래 항목을 확인하세요. 이 단계에 시간을 투자할수록 학습 자료의 품질이 높아집니다.

**구조 점검:**
- [ ] 전체 Chapter 수가 12~25개 범위인가?
- [ ] 각 Part의 Chapter 수가 3~6개로 균형이 잡혀 있는가?
- [ ] 각 Chapter의 Section 수가 2~8개 범위인가?
- [ ] 각 Chapter의 예상 학습 시간이 2~4시간으로 적절한가?

**선수지식 & 개념 폐쇄성 점검:**
- [ ] 각 Chapter의 "선수 지식"이 합리적인가? (순환 의존은 없는가?)
- [ ] 각 Chapter의 "도입 개념"이 명시되어 있는가?
- [ ] 특정 Chapter를 학습할 때, 선수지식 Chapter들의 도입 개념만으로 충분히 이해 가능한가?
- [ ] 학습 로드맵의 의존 관계 화살표가 누락 없이 표시되어 있는가?

**내용 점검:**
- [ ] 해당 주제를 마스터하기 위해 빠진 핵심 내용은 없는가?
- [ ] 중복되거나 너무 비슷한 Chapter가 있지는 않은가?
- [ ] 난이도 표기가 적절한가?
- [ ] 실용적 활용(실습, 프로젝트 등)이 충분히 포함되어 있는가?

### 3.4 커리큘럼 수정 요청 예시

같은 대화에서 수정을 요청합니다:

- "Chapter 5와 6의 순서를 바꿔주세요. 5의 내용이 6을 선수지식으로 요구하지 않으니까요."
- "Part 3에 실전 프로젝트 Chapter를 하나 추가해주세요."
- "Chapter 8의 Section이 너무 많아요. 두 개의 Chapter로 분리해주세요."
- "Chapter 3의 난이도가 ⭐⭐인데, 내용을 보니 ⭐⭐⭐이 맞는 것 같아요."
- "전체적으로 Chapter 수가 너무 많아요. 비슷한 주제의 Chapter를 합쳐주세요."
- "Chapter 7에서 도입 개념으로 XX가 빠져 있어요. 추가해주세요."
- "Chapter 10의 선수지식에 Chapter 6이 없는데, 6의 YY 개념이 필요할 것 같아요."

### 3.5 커리큘럼 저장

최종 확정된 커리큘럼을 .md 파일로 저장합니다.

권장 파일명: `[주제]_curriculum.md`  
예: `python_programming_curriculum.md`, `linear_algebra_curriculum.md`

## 4. Step 2: 프로젝트 세팅

### 4.1 프로젝트 생성

1. Claude에서 **새 프로젝트**를 만듭니다.
2. 프로젝트 이름을 설정합니다. (예: `📘 파이썬 프로그래밍 학습`)

### 4.2 프로젝트 지침 설정

**프로젝트 지침(Instructions)** 영역에 `02_study_material_prompt.md`의 **전체 내용**을 붙여넣습니다.

이렇게 하면 프로젝트 내의 모든 대화에서 이 지침이 자동으로 적용됩니다.

> ⚠️ **프로젝트 지침 글자 수 제한**: Claude 프로젝트의 지침(Instructions) 영역에는 글자 수 제한이 있습니다. 02_study_material_prompt.md는 이 제한 내에서 동작하도록 설계되어 있지만, 분야별 추가 지시(7.1 참조)를 붙일 때는 제한을 초과하지 않도록 주의하세요. 만약 초과한다면 추가 지시를 프로젝트 파일(Knowledge)로 첨부하는 방법을 사용하세요.

### 4.3 커리큘럼 파일 첨부

**프로젝트 파일(Knowledge)** 영역에 저장해둔 커리큘럼 .md 파일을 업로드합니다.

이렇게 하면 Claude가 매번 커리큘럼 전체를 참고하면서 학습 자료를 생성합니다.

### 4.4 세팅 완료 후 구조

```
📁 프로젝트: 파이썬 프로그래밍 학습
├── 📋 지침: 02_study_material_prompt.md 내용
├── 📎 파일: python_programming_curriculum.md
├── 💬 대화 1: Chapter 1 학습 자료
├── 💬 대화 2: Chapter 2 학습 자료
├── 💬 대화 3: Chapter 3 학습 자료
└── ...
```

## 5. Step 3: 상세 학습 자료 생성

### 5.1 기본 사용법

프로젝트 내에서 **새 대화**를 시작하고, 다음과 같이 요청합니다:

```
Chapter 1의 학습 자료를 작성해주세요.
```

이것만으로 충분합니다. 프로젝트 지침과 커리큘럼이 이미 세팅되어 있으므로, Claude가 해당 Chapter의 학습 목표, Section 구성, 선수 지식 등을 모두 참고하여 작성합니다.

### 5.2 출력이 잘리는 경우

학습 자료의 분량이 많아 출력이 중간에 끊길 수 있습니다.

**기본 대응:**
- "계속해주세요" 또는 "이어서 작성해주세요"라고 입력합니다.

**분할 요청 전략 (Section이 많은 Chapter에 권장):**

한 Chapter의 Section 수가 5개 이상이면, Part 단위로 나누어 요청하는 것이 품질 면에서 더 좋을 수 있습니다:

```
1번 대화:
  "Chapter 5의 Part A와 Part B를 작성해주세요."
  → (출력이 끊기면) "계속해주세요."
  → Part A + Part B 완성

2번째 메시지:
  "이어서 Part C, D, E를 작성해주세요."
  → Part C + D + E 완성
```

또는 Part B만 별도로 요청할 수도 있습니다:
```
"Chapter 5의 Part B (본문)만 작성해주세요."
```

### 5.3 보충 및 추가 요청

같은 대화에서 추가 요청이 가능합니다:

- "Section 3.2의 직관적 이해 부분을 더 자세히 설명해주세요."
- "Exercise에서 실무 관련 문제를 2개 더 추가해주세요."
- "Practice가 너무 쉬운 것 같아요. 조금 더 생각이 필요한 문제로 바꿔주세요."
- "Deep Dive 부분에서 수학적 증명을 추가해주세요."
- "Problem을 3개 더 만들어주세요."
- "Section 3.4에서 아직 안 배운 개념이 사용된 것 같아요. 수정해주세요."

### 5.4 학습 자료 저장

권장 파일명 형식: `Ch[번호]_[제목].md`  
예: `Ch03_제어문.md`, `Ch07_고유값과_고유벡터.md`

> 💡 **팁**: 모든 Chapter 파일을 하나의 폴더에 모아두면 전체 교재처럼 사용할 수 있습니다.

## 6. 실전 예시 — 처음부터 끝까지

### 📍 예시: "확률과 통계" 학습 자료 만들기

#### Step 1: 커리큘럼 생성 (일반 대화)

01_curriculum_prompt.md를 복사하고 `[학습 주제]`를 수정하여 전송:

> (프롬프트 내용 전체)  
> ... 이제 **통계학 (대학 수준, 이공계 전공자 기준)**에 대한 커리큘럼을 설계해주세요.

Claude가 커리큘럼을 생성합니다. 검토 체크리스트(3.3절)를 활용하여 검토 후 수정 요청:

> Chapter 4에 "베이즈 정리"를 별도 Section으로 추가해주세요.  
> Part 4에 "시뮬레이션과 몬테카를로 방법" Chapter를 추가해주세요.  
> Chapter 7의 도입 개념에 "표본분포"가 빠져 있어요. 추가해주세요.  
> 수정된 전체 커리큘럼을 다시 출력해주세요.

최종 커리큘럼을 `statistics_curriculum.md`로 저장합니다.

#### Step 2: 프로젝트 세팅

1. 새 프로젝트 `📘 확률과 통계 학습` 생성
2. 프로젝트 지침에 02_study_material_prompt.md 내용 붙여넣기
3. 프로젝트 파일에 `statistics_curriculum.md` 업로드

#### Step 3: Chapter 1 학습 자료 생성 (프로젝트 내 대화)

> Chapter 1의 학습 자료를 작성해주세요.

출력이 끊기면:
> 계속해주세요.

만족스러우면 `Ch01_경우의_수.md`로 저장합니다.

#### Step 4: 나머지 Chapter 반복

프로젝트 내에서 각각 새 대화를 시작하여 Chapter 2, 3, ... 순서대로 생성합니다.

## 7. 고급 활용법

### 7.1 분야별 커스터마이징

프로젝트 지침의 끝에 분야 특화 지시를 추가할 수 있습니다. 프로젝트 지침의 글자 수 제한에 주의하세요. 초과할 경우 별도 .md 파일로 만들어 프로젝트 파일(Knowledge)에 첨부하세요.

**프로그래밍 분야 추가 지시:**
```
추가 규칙:
- 모든 코드 예시는 실행 가능한 완전한 코드로 작성하세요.
- 각 Section에 "직접 해보기(Try It Yourself)" 코너를 추가하세요.
- Practice/Exercise에서 실제로 코드를 작성하는 문제를 포함하세요.
```

**수학 분야 추가 지시:**
```
추가 규칙:
- 모든 정리(Theorem)에는 증명을 포함하세요.
- 기하학적 직관을 위한 그래프/도형 설명을 텍스트로 상세히 묘사하세요.
- Problem에는 증명 문제를 최소 1개 포함하세요.
```

**인문/사회 분야 추가 지시:**
```
추가 규칙:
- 주요 학자와 그들의 핵심 주장을 반드시 소개하세요.
- 서로 다른 관점/학파의 비교를 포함하세요.
- Exercise에는 에세이 형태의 문제를 포함하고, 모범 답안의 논리 구조를 분석하세요.
```

### 7.2 이어지는 과정 연결

하나의 과정을 완료한 후 후속 과정을 만들 때, 커리큘럼 프롬프트에 다음을 추가합니다:

```
참고: 이 과정은 아래 과정의 후속 과정입니다. 
아래 과정의 마지막 Chapter 수준에서 시작해주세요.

<previous_course>
(이전 과정의 커리큘럼 붙여넣기)
</previous_course>
```

### 7.3 복습 자료 생성

프로젝트 내에서 복습 자료를 요청할 수도 있습니다:

```
Chapter 1~5의 종합 복습 자료를 만들어주세요.

요청사항:
- 핵심 개념을 통합 정리
- 개념 간 연결 관계 중심으로 정리  
- Chapter를 넘나드는 통합 문제 5개 (Exercise 수준)
- Chapter를 넘나드는 심화 문제 3개 (Problem 수준)
```

### 7.4 학습 자료의 난이도 조절

학습 자료 생성 시 난이도를 조절하고 싶다면 요청에 명시합니다:

```
Chapter 3의 학습 자료를 작성해주세요.
단, 대학원 진학을 준비하는 학부 4학년 수준에 맞춰주세요.
Problem은 대학원 입시 수준으로 출제해주세요.
```

## 8. 트러블슈팅 & 팁

### 흔한 문제와 해결법

| 문제 | 원인 | 해결법 |
|------|------|--------|
| 설명이 너무 간략함 | 분량 기준이 무시됨 | "각 Section을 더 상세하게 작성해주세요. 직관적 이해 부분을 특히 풍부하게 써주세요." |
| 문제가 너무 쉬움/어려움 | 난이도 판단 오차 | "Problem의 난이도를 대학원 입시 수준으로 올려주세요" 등 구체적 기준 제시 |
| Part A~E 구조가 지켜지지 않음 | 긴 출력에서 구조를 잊음 | "Part A~E의 구조를 반드시 지켜주세요. 현재 Part B의 Section N.M부터 이어서 작성해주세요." |
| 중간에 출력이 끊김 | 토큰 제한 | "계속해주세요" 입력. 잦은 끊김 시 분할 요청 전략(5.2) 사용 |
| 이전 Chapter 내용과 겹침 | Chapter 구분이 모호 | 커리큘럼에서 Chapter 간 경계를 더 명확히 수정 |
| 예시가 부족하거나 비현실적 | 구체적 요구 부족 | "실무에서 자주 접하는 상황을 예시로 들어주세요" 등 방향 지정 |
| 커리큘럼의 Section과 다른 Section이 생성됨 | 커리큘럼 참조 실패 | "커리큘럼의 Chapter N에 명시된 Section 목록을 정확히 따라주세요" |
| **아직 안 배운 개념이 설명에 등장함** | 선수지식 범위 위반 | "Section X.Y에서 [개념]은 아직 배우지 않은 내용입니다. 이 시점에서 학습자가 아는 개념만으로 다시 설명해주세요." |
| **문제에 미학습 개념이 필요함** | 문제 출제 시 범위 초과 | "이 문제는 Chapter N의 개념이 필요한데, 아직 배우지 않았습니다. 현재까지 배운 범위 내에서 풀 수 있도록 수정해주세요." |

### 품질 향상 팁

1. **커리큘럼에 투자하세요**: 커리큘럼이 잘 설계되어야 학습 자료의 품질도 높아집니다. 커리큘럼 수정에 시간을 아끼지 마세요. 특히 "도입 개념"과 "선수지식"이 정확해야 학습 자료의 개념 범위가 올바르게 유지됩니다.

2. **피드백을 구체적으로**: "더 좋게 해주세요"보다 "Section 3.2에서 행렬 곱셈의 기하학적 의미를 그림으로 설명하듯 텍스트로 묘사해주세요"가 훨씬 효과적입니다.

3. **한 번에 하나의 Chapter**: 여러 Chapter를 한 번에 요청하면 품질이 떨어집니다. Chapter 하나씩 새 대화에서 생성하세요.

4. **생성 후 직접 학습해보세요**: 직접 읽고 학습해보면 부족한 부분이 드러납니다. 특히 "이 설명을 이해하려면 아직 안 배운 것을 알아야 하나?"를 체크하세요. 발견한 부족한 점은 같은 대화에서 보충 요청하세요.

5. **문제부터 풀어보세요**: Practice → Exercise → Problem을 직접 풀어보고, 해설이 충분한지 확인하세요.

6. **첫 Chapter로 테스트하세요**: Chapter 1의 학습 자료를 먼저 생성해보고, 스타일과 깊이가 마음에 드는지 확인한 뒤 나머지를 진행하세요. 불만족스러운 부분이 있으면 프로젝트 지침에 추가 규칙을 넣어서 이후 Chapter에 반영되게 하세요.

## 9. 용어 체계 정리

### 구조 계층

| 계층 | 한국어 | 영어 | 설명 | 비유 |
|------|--------|------|------|------|
| Level 0 | 과정 | Course | 전체 학습 주제 | 교과서 한 권 |
| Level 1 | 편 | Part | 대분류 | 교과서의 "제1편" |
| Level 2 | 장 | Chapter | 학습 자료 요청의 기본 단위 | 교과서의 "제3장" |
| Level 3 | 절 | Section | Chapter 내 세부 주제 | 교과서의 "3.2절" |
| Level 4 | 항 | Subsection | Section 내 더 세부 주제 (필요시) | 교과서의 "3.2.1항" |

### 난이도 기준

| 난이도 | 기준 |
|--------|------|
| ⭐ | 특별한 배경 없이 이해 가능 |
| ⭐⭐ | 약간의 집중이 필요 |
| ⭐⭐⭐ | 여러 개념을 연결해야 함 |
| ⭐⭐⭐⭐ | 추상적 사고나 복잡한 논리 필요 |
| ⭐⭐⭐⭐⭐ | 전문가 수준 |

> **중요**: 난이도는 커리큘럼의 배치 순서와 무관합니다. 순서는 선수지식 의존성에 의해 결정되고, 난이도는 각 Chapter의 고유 속성입니다.

### 문제 유형

| 유형 | 한국어 | 난이도 | 목적 | 특징 |
|------|--------|--------|------|------|
| Practice | 연습 문제 | ⭐~⭐⭐ | 개별 개념 확인 | 1개 개념, 즉각 피드백 |
| Exercise | 응용 문제 | ⭐⭐~⭐⭐⭐ | 개념 통합 적용 | 2개+ 개념, 시나리오 기반 |
| Problem | 심화 문제 | ⭐⭐⭐⭐~⭐⭐⭐⭐⭐ | 창의적 문제 해결 | 열린 결말, 실전 수준 |

### 학습 자료 구조 (Chapter 내부)

| 파트 | 명칭 | 포함 내용 |
|------|------|-----------|
| Part A | 도입부 | 표지, 학습 목표, 핵심 질문, 개념 지도 |
| Part B | 본문 | Section별: 직관 → 핵심 설명 → 예시 → Deep Dive → 주의사항 → 요약 |

| Part D | 문제 풀이 | Practice → Exercise → Problem (3단계) |
| Part E | 마무리 | 다음 단계 안내, 추가 자원 |
